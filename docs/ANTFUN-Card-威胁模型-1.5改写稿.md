## 1.5　卡组克隆备份(取代助记词的核心设计)

**产品目标**:让用户不再需要手动备份助记词;并支持在 NFC 消磁 / 卡片丢失后,把私钥迁移到一套全新卡组。

**形态**:一套 N 张卡(建议 1 主 + 2 备),互为完整副本;丢失任意一张,用另一张继续。**主密钥以 512-bit 种子形态在卡间以密文复制,助记词从不存在**(见 §0 无助记词红线)。

**安全收益是决定性的**:全行业针对用户的终极攻击都是「请输入你的 24 个助记词」——用户手里根本没有助记词,这个攻击面直接消失。

### 克隆协议(A=源卡,B=目标卡;既可同组备份,也可跨组迁移到新卡)

```
① B 卡输出:自身 DAK 设备证书(authentikey 公钥 + 我方 CA 签名)+ 卡内新生成 nonce_B
② App 转发①给 A(仅搬运,因有 CA 签名而不可篡改)
③ A 卡在【卡内】校验,任一不过即拒:
     a. 用内置 CA 公钥验证 B 的 DAK 证书签名 → 确认 B 是「我方出厂真卡」(非任意野公钥)
     b. 校验 nonce_B 新鲜 → 防重放
     + PIN 已验证
④ A 卡以【临时】ECDH(临时私钥 e_A × B 的 authentikey 公钥)导出共享密钥;
     KDF = HKDF-SHA256(ECDH ‖ nonce_B);AEAD(AES-CCM,密钥由 HKDF 派生)加密 512-bit 主种子;输出 密文 ‖ e_A 公钥
⑤ App 转发密文给 B;B 卡以 authentikey 私钥 + e_A 公钥 复算密钥、校验 nonce_B、GCM 解密 → 卡内装载
⑥(可选·封印模式)用户若选「备份完成、永不再迁移」→ 置 OTP 熔断位 → 该卡克隆指令永久不可用
```

### 关键安全性质

- **卡厂无法克隆**:套内联结靠各卡**卡内自生成**的 authentikey 公钥(公开信息);卡厂不持有任何可解密的私钥。DAK 证书由我方 HSM 的 CA 签发,卡厂只注入密文包。
- **接收方认证在卡内,不在 App**(相对原稿的关键修正):第 ③a 步由 A 卡用内置 CA 公钥验证 B 的证书。**木马 App 无法把主种子骗给一个野公钥**——野公钥没有我方 CA 签名,A 卡直接拒绝。攻击者若想当合法目标,只能用**一张自己购买的真卡**,且导入后须**物理持卡**方能动用 → 攻击退化为「需实体卡 + 劫持一次克隆时刻」的可接受档。
- **临时 ECDH + nonce**:每次会话密钥不同,捕获的旧密文不可重放(相对原稿静态方案的加固)。
- **PIN 强制**:卡组模式下 PIN 为必选项;PIN 错误计数锁死。

### 默认可迁移 vs 可选封印(相对原稿的放宽)

- **默认:克隆通道长期可用(PIN + 卡内 DAK 认证门控)**。合法目标是任何一张我方真卡,因此支持「源卡 → 全新目标卡」的迁移,解决 NFC 消磁 / 丢卡后换卡的刚需。
- **可选:封印模式**。面向「已备份完、要极致安全、永不再迁移」的高阶用户,自选置 OTP 熔断位(硬件一次性、独立于 factory-reset、不可逆),此后该卡永不可再作为克隆源/目标。这保留了原稿「完成即熔断」的极致安全,但改为**用户自选**而非强制。

### 代价与告知义务

- 全套卡同时丢失 = 资产永久不可恢复。产品必须强制告知并引导**分地点保管**。
- **放宽熔断带来的新残余风险**(须在产品与审计中正视):克隆通道长期存活,意味着一次 PIN 已验证的会话中,恶意 App 可能静默发起一次「克隆到攻击者自购真卡」。缓解:①克隆作为独立高敏操作,要求**每次单独确认 / 重新输入 PIN**,不复用普通签名会话的 PIN 状态;②可加克隆冷却期 / 次数告警;③高价值用户引导启用封印模式。面向高阶用户可提供 App 端加密备份作为第三条路径(可选,非默认)。
